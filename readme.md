# komunalka
__Описание:__ сервис учета коммунальных платежей  

__Особенности версий:__ разные подходы к обработке адресов

## версия 1
Простая структура из физических файлов в корне проекта.
Версия характерна адресами "/create.php", "/update.php"

## версия 2
Адреса формируются из физических папок одноименных с адресом. 
Например, в корне проекта существует папка "create" внутри которой существует index.php, это будет означать, что существует адрес "/create"
В этой версии можно наращивать адреса путем добавления подпапок с index.php внутри.
Например, для адреса "/create/test1" нужно в папке "create" создать вложенную папку "test1" (c index.php внутри).

## версия 3
Единая точка входа.
Адреса и физические файлы и папки не имеют прямой взаимосвязи. Все адреса формируются виртуально в "роутере" (блок кода, отвечающий за обработку адресов).
